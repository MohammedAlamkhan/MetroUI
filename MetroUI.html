required libraries

The JavaScript components of MetroUI are based on jQuery, so you will need to include it before anything else. Then you can include MetroUI's JavaScript file. And don't forget to include the CSS file!

Code - html

	<link rel="stylesheet" type="text/css" href="css/MetroUI.css">
	
	<script type="text/javascript" src="http://code.jquery.com/jquery-1.11.0.min.js"></script>
	<script type="text/javascript" src="javascript/MetroUI.js"></script>

the metro font

As the original Metro fonts, called 'Segoe UI' for Windows 8 and 'Segoe WP' for Windows Phone 8, are distributed via licenses, MetroUI can't just include them so everyone gets his hands on it.
Instead, MetroUI uses a font called 'Open Sans' in various font weights, which really looks a lot like Segoe. And the best of it: it is distributed for free via Google Fonts, so we won't get into licensing troubles.

If you want Open Sans on your site too, just include the following lines in the <head></head> region of your HTML and you're good to go. Just make anything use "font-family: 'Open Sans', sans-serif" and one of the font-weights.

Code - HTML:

		<link href='http://fonts.googleapis.com/css?family=Open+Sans:400,600,700,300' rel='stylesheet' type='text/css'>

touch support

MetroUI and its components support touch based input. Touch based input removes the annoying 300ms delay on touch devices, so actions are performed instantaneously. It is achieved with two variables setting an event based on the users screen.

Code - JavaScript:

	var touchEventStart = (('ontouchend' in window)) ? 'touchstart' : 'mousedown';
	var touchEventEnd = (('ontouchend' in window)) ? 'touchend' : 'click';


These lines are included within MetroUI's Javascript component, so you don't have to include them yourself anywhere. They can also be used for anything you would like to add touch support to.

the 'create' function

Every MetroUI element is initialized with the 'create' function. It is making any raw element shown on this page (except for lists and accent lists right now) a working MetroUI element. Using it is extremely simple! Here is an example:

Code - JavaScript:

	MetroUI.create({
		type: "switch",
		objID: $('#switch1'),
		id: "switch1",
		enabled: true,
		startSwitched: false
	});


This example makes a switch out of the jQuery object "$('#switch1)" and assigns it the "data-id=switch1". It is also set to be unswitched at start. Let's have a look on the options you have when creating a MetroUI element:

argument type options default setting notes
type String "button", "switch", "list", "accentList" undefined "list", "accentList" not supported yet
objID jQuery Object none undefined Can be jQuery .class instead of #id if unique
id String none undefined Should be equal to the specified jQuery object
active boolean true, false true Is the element activated on start or not?
startSwitched boolean true, false false Only applicable to switches
callback function none none What an element has to do after being used.


Every argument that defaults to "undefined" is required, others are optional.

Please note that the types "list" and "accentList" aren't implemented yet, so trying to create elements of those types will be unsuccessful.

buttons

Coming now to the real UI elements, starting with Buttons. Buttons are used for temporary interaction, whether in forms or just to run a JavaScript function.
They can easily be inserted using code as shown below.

Example:

Use the button to switch between dark and light backgrounds.

Code - HTML:

	<button id="button1" type="button">Button</button>
	<button id="button2" type="button">Disabled</button>



Code - JavaScript:

	MetroUI.create({
		type: "button",
		objID: $('#button1'),
		id: "button1",
		callback: function() { setTimeout(function() { $('body').toggleClass("darkBG lightBG") }, 20) }
	});
	MetroUI.create({
		type: "button",
		objID: $('#button2'),
		id: "button2",
		active: false
	});

On buttons, a 20ms delay is needed to perform an action. If it has no delay, actions are likely to be performed multiple times on touch based devices.

switches

Switches are used, unlike buttons, for long-term interaction, meaning they can change settings for a longer period than buttons.

Example:


Code - HTML:

	<div class="metro checkbox" id="switch1">
	    <div class="innerCheckbox"></div>
	</div>
	
	<div class="metro checkbox checked" id="switch2">
	    <div class="innerCheckbox"></div>
	</div>

	<div class="metro checkbox disabled" id="switch3">
	    <div class="innerCheckbox"></div>
	</div>

theming support

Current accent color has been set to Amber for presentation purposes.

More colors coming with MetroJS implementation, meaning every color of Windows Phone 8 will be supported. Besides that, every UI element supports dark and light backgrounds.
