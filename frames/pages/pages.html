Description

Pages are the most substantial part in MetroUI. They contain everything, from simple Form elements to more advanced things like Grids and Hubs.

In MetroUI, Pages have a name for identification (eg. in Callbacks) and a category, that is shown in the Navigation Bar (if available).

When changing to another page, the new page is made the current page that replaces the old page. They can also be loaded from external files, which happens automatically when adding a link to the content.

Pages are stored in a single Page container, so for external files a minimal layout is required (shown in the Code section).

Attributes & Classes
Attribute Description
data-page Specifies the page title. - Required for Navigation and Page Callbacks
data-category Specifies the page category. - Required for Navigation Bar (if available)
